## Проект
**Panamera Studio** — живой веб-конструктор тюнинга Porsche Panamera (972 / G3,
MY2024+). Открывается на телефоне: фото залипает сверху, опции под ним, фото
меняется сразу; билд сохраняется в URL-hash и шлётся в Telegram.

- Live: https://alexautorental.github.io/panamera-studio/
- Репозиторий: `alexautorental/panamera-studio`
- Деплой: GitHub Actions выкладывает `public/` при пуше в основную ветку (~30 сек).
- Главный код: `public/index.html` (canvas live-движок), данные —
  `public/tuning-assets/data/`, инструменты — `tools/`.
- Подробности: корневой `README.md`, `SOURCES.md`, `QA.md`.

## О чём была эта сессия
Вопрос владельца: как сделать, чтобы **второй личный аккаунт** Claude Code видел
те же проекты и ту же переписку.

**Вывод:**
1. Чаты/сессии привязаны к аккаунту и **между аккаунтами не переносятся** —
   штатной функции нет.
2. Реально синхронизируется только **git-репозиторий** → код общий, если у обоих
   аккаунтов есть доступ к репозиторию на GitHub.
3. Чтобы сохранить и переписку — историю сессии **выгрузили в файлы и
   закоммитили** в `docs/claude-sessions/` (эта папка). Новый аккаунт читает их и
   восстанавливает контекст «для себя» (не как визуальный чат — это невозможно).

## Что делать новому аккаунту
1. Дать второму аккаунту доступ к репозиторию `alexautorental/panamera-studio` на
   GitHub (коллаборатор).
2. Открыть репозиторий новой сессией на claude.ai/code.
3. Попросить: «Прочитай `docs/claude-sessions/CONTEXT.md` и продолжим».
