# Claude Code — история сессий (перенос между аккаунтами)

Эта папка нужна, чтобы **другой аккаунт Claude Code** (второй личный аккаунт того
же владельца) мог продолжить работу над проектом Panamera Studio с сохранением
контекста переписки.

## Почему это здесь, а не «в чатах»

Чаты (сессии) Claude Code **привязаны к конкретному аккаунту** и не переносятся
между разными учётками — такой функции нет. Единственное, что реально
синхронизируется между аккаунтами, — это **git-репозиторий**. Поэтому историю
переписки мы выгружаем в файлы и коммитим сюда: код и контекст едут вместе с
репозиторием.

## Что в папке

| Файл | Что это |
|------|---------|
| `session-<id>.md` | Читаемая история диалога (только текст, без внутренних служебных данных). |
| `session-<id>.raw.jsonl` | Сырой транскрипт сессии — полная точность для машинного восстановления. |
| `CONTEXT.md` | Краткая сводка: состояние проекта и о чём была договорённость. Начинать чтение отсюда. |

## Как восстановить контекст на новом аккаунте

1. Войдите вторым аккаунтом на **claude.ai/code**.
2. Убедитесь, что у этого аккаунта есть доступ к репозиторию на GitHub
   (`alexautorental/panamera-studio`). При необходимости добавьте аккаунт в
   коллабораторы репозитория на GitHub.
3. Создайте новую сессию Claude Code с этим репозиторием.
4. Скажите Claude что-то вроде:
   > «Прочитай `docs/claude-sessions/CONTEXT.md` и историю в
   > `docs/claude-sessions/`, восстанови контекст и продолжим.»

Claude прочитает файлы и «войдёт в курс дела» — визуально старый чат не появится
(это невозможно между аккаунтами), но контекст будет восстановлен для дальнейшей
работы.

## Ограничение, о котором важно знать

Сюда попадает **только та сессия(и), которую выгрузили вручную из своего
окружения**. Облачные контейнеры Claude Code эфемерны и создаются с нуля, поэтому
из одной сессии не видно транскрипты других, ранее существовавших чатов. Чтобы
сохранить ещё какие-то прошлые чаты — их нужно выгрузить так же (из той сессии) и
добавить в эту папку.
