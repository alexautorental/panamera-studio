- Суппорты жёлтый/зелёный/синий/в цвет кузова (кроме PCCB/PSCB) — кастом-окрас → needs-verification.
- PPF — плёнка vehicle-agnostic, но выкройку под 972 у установщика проверить.
- Тонировка тёмная — проверить локальную легальность (перед лобовое/передние).
- Performance Stage 1 — только инфо-раздел с предупреждением (гарантия/экология/ECU/гибрид), НЕ визуализируется.

## 🟡 Слабые визуалы — где Codex должен заменить на лучшее
Сейчас фото меняет **только цвет + финиш плёнки + свет**. Остальной тюнинг показан схемой/бейджами, НЕ на фото.
Для «вау» нужны реальные слои (по приоритету):
1. **Диски** — нет визуальной смены диска на фото. Нужны PNG-оверлеи дисков по каждому ракурсу (маска области колеса) или отдельные рендеры. Папка готова: `public/tuning-assets/overlays/wheels/`.
2. **Обвес / карбон / widebody** — нет на фото. Нужны выровненные PNG-оверлеи по ракурсам (`overlays/bodykits/`, `overlays/carbon/`) или полноценные рендеры комплектов.
3. **Тонировка** — сейчас только чип. Нужна маска стёкол, чтобы реально затемнять окна (`overlays/tint/`).
4. **Суппорты** — слишком мелкие на общих планах; нужен макро-кадр колеса или crop-оверлей.
5. **Посадка/занижение** — нельзя занизить 2D-фото; нужен отдельный рендер низкой посадки.
6. **Салон** — нет фото интерьера; нужны реальные кадры салона по цветам (`overlays/interior/`).
7. **Финиш matte/satin/stealth** — сейчас CSS-аппроксимация фильтром; реальный матовый рендер точнее.

**Рекомендация Codex:** держать тот же honest-подход (реальные рендеры/оверлеи, без фейк-«замазки»);
концепты оставлять помеченными. Структура данных (`tuning-options.json`, поле `assets`) уже готова принять пути оверлеев.
